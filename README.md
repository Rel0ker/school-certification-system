# Система промежуточной аттестации

## Репозиторий

- Клон: `git clone https://github.com/Rel0ker/school-certification-system.git`
- После клона, один раз: `npm install && npm run update-vendor` — в папку `vendor/` копируются `sql-wasm.js` и **`sql-wasm.wasm`** (нужны для работы в браузере), если их ещё нет.

## Как устроена «единая» система

- Всё в одной папке: HTML, CSS, JS, SQLite в WebAssembly (файл `vendor/sql-wasm.wasm`) и `db.js`.
- **Node.js и отдельный бэкенд не нужны.** Данные хранятся в браузере: движок **SQLite (sql.js)** + бинарь базы в **IndexedDB** (и дублируется JSON в `localStorage`).

## Запуск (один раз, одна команда)

Из этой папки в терминале:

```bash
python3 start.py
```

Скрипт поднимет HTTP на **всех сетевых интерфейсах** (удобно для доступа с телефонов и других ПК по Wi‑Fi) и печатает **три готовых ссылки** — `127.0.0.1`, `localhost` и **IP в локальной сети** (и при необходимости запасные IP). Сразу копируйте третью ссылку и отправляйте коллегам. Также пытается открыть браузер на этой же машине.

Порт: переменная окружения `PORT`, например `PORT=3000 python3 start.py`.

Альтернатива без `start.py` (то же самое вручную):

```bash
python3 -m http.server 8080
```

**Не открывайте `index.html` как файл (file://)** — загрузка WebAssembly с диска во многих браузерах заблокирована. Нужен обычный `http://` адрес, как у команды выше.

## Бэкап

- **Экспорт JSON** (блок в интерфейсе) — для переноса на другой компьютер или в другой браузер.
- Все данные привязаны к **этому** браузеру и **этому** сайту (домен/порт). Смена браузера = пустая база, пока не импортируете JSON.

## Обновление движка SQL (только для разработчика)

Если стоит Node:

```bash
npm install
npm run update-vendor
```

Файлы `vendor/sql-wasm.js` и `vendor/sql-wasm.wasm` обновятся; без npm можно скачать их вручную с [sql.js.org](https://sql.js.org/).

## Формат XML

- Список учеников: колонки «Класс», «ФИО ребенка».
- Нагрузка: «Класс», «Предмет», «Учитель» (Excel XML 2003).
